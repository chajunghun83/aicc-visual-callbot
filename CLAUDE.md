- Vue 3 + Vite
- 중앙 상태 관리: Pinia (스토어에 "진입 컨텍스트 + 슬롯 상태 + 시나리오 스크립트" 보관)
- 목업 데이터: 중앙 집중 스토어(필요 시 MSW). 실제 API와 동일한 형태로 구성해 향후 백엔드 전환을 쉽게.
- 스타일: Tailwind CSS (조정 가능)
- 대상: **모바일폰 세로 기준 설계**, 태블릿 대응(콘텐츠 최대 폭 제한 + 여백 확장), **PC 비대상**

## 아키텍처 원칙

- **대화 주도(Conversation-led):** 봇이 단계에 맞는 컴포넌트를 능동적으로 노출한다. 사용자가 메뉴를 탐색하는 메뉴 주도 방식으로 만들지 않는다.
- **고정 컴포넌트 팔레트 + 상태(서버) 주도 노출:** 사전 정의된 컴포넌트 중 상태에 맞는 것을 골라 배치한다. **LLM이 UI를 자유롭게 생성하는 방식 금지.**
- **음성·터치 통합 슬롯:** 모든 입력형 컴포넌트는 대화 상태의 특정 슬롯을 채우는 동작으로 통일한다.
- **단일 프론트엔드 + 채널 분기:** 전화/앱 모두 동일한 앱으로 구동. 채널 차이는 진입 파라미터(예: `?channel=phone|app`) + 기능 플래그로 분기한다. **화면을 채널별로 복제하지 않는다.**
- **컴포넌트 채널 비종속:** 진입 컨텍스트(채널·세션·고객정보·시나리오)를 props/스토어로 주입받게 설계한다. **컴포넌트에 채널·시나리오를 하드코딩하지 않는다.**

## 화면 구조 (5-zone)

1. **TrustHeader** — 브랜드·안심마크·통화 상태·타이머
2. **WaveIndicator** — AI 파동(대기/듣는 중/말하는 중 상태 표시)
3. **메인 콘텐츠 영역** — 봇이 팔레트 컴포넌트를 갈아 끼우는 가변 영역. '한 화면 한 과업'
4. **LiveCaption** — 실시간 자막(STT/TTS)
5. **QuickActionBar** — 전역 액션(상담사 연결 / 음성 전환 / 캡처 전송), 항상 같은 자리

큰 터치 타깃·큰 글씨·고대비·적은 정보량을 기본으로 한다.

## 컴포넌트

- **셸(항상 존재):** TrustHeader, WaveIndicator, LiveCaption, QuickActionBar
- **콘텐츠 팔레트:** InfoMessageCard, ChoiceButtons, DataCard, FormInput, IdentityVerify, PaymentForm, FileUpload, AnalysisResult, SignaturePad, ResultCard
- **1차 구현 우선:** InfoMessageCard, ChoiceButtons, IdentityVerify, DataCard, FileUpload, AnalysisResult

## 1차 데모 시나리오 (내 보유 종목 조회)

- **채널/방향:** 전화 인바운드
- **흐름:** ① 진입·환영(InfoMessageCard) → ② 종목 선택/검색(ChoiceButtons/FormInput) → ③ 본인인증: 휴대폰 뒷자리 4자리(IdentityVerify) → ④ 보유 종목 정보: 현재가·등락률·평가손익(DataCard) → ⑤ 와우 연출: 앱 오류 캡처 분석(FileUpload→AnalysisResult) → ⑥ (선택) 후속 액션
- **캡처 분석 = 스크립트(A안):** 실제 Vision 인식 없음. 이미지를 올리면 사전 정의된 결과를 표시하고, "화면 분석 중..." 로딩 연출을 넣는다. 결과는 이미지의 구체적 요소를 짚는 문구로 작성.
- **TTS:** 봇 음성 안내에 TTS 적용을 전제로 한다(방식은 추후 결정). 음성 재생 시 WaveIndicator '말하는 중' + 자막 동기화.

- **이모지 사용 금지.**

- **PC 전용 레이아웃 만들지 않는다**(모바일폰·태블릿만).
- **백엔드/실제 API 호출 금지**(목업 데이터만 사용).

## 작업 방식

- **개발 서버 자동 새로고침 금지:** Vite HMR을 끈다(`vite.config.js`의 `server.hmr: false`). 코드 변경 시 브라우저가 자동으로 새로고침되지 않으며, **새로고침은 사용자가 직접 한다.** 이 설정을 임의로 되돌리지 않는다.
- **빌드 순서:** 시작 화면(전화/앱) → 5-zone 셸 → 1차 시나리오 end-to-end → 팔레트 확장.
- 목업은 단계별 시나리오 스크립트(봇 발화 → 노출 컴포넌트 → 사용자 액션 → 다음 단계)를 담아 데모가 단계별로 재생되게 한다.
- 커밋/완료 전, 이모지·민감정보 직접 수집 여부를 점검한다.
- **Git 커밋·푸시는 사용자가 명시적으로 요청할 때만 수행한다.** 코드 변경을 마쳐도 자동으로 커밋/푸시하지 않는다(요청 시 `add → commit → push`로 반영).
