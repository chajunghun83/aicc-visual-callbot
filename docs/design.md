# 디자인 가이드 (Design Guide): AICC 보이는 콜봇

> PRD(`docs/Visual Callbot_PRD.md`)에서 디자인 관련 요소를 분리해 정리한 문서.
> 기능·시나리오 사양은 PRD를, 시각·톤·스타일 결정은 이 문서를 단일 진실 출처로 삼는다.
> 세부 색·수치는 빌드하면서 조절하며, 여기서는 방향과 원칙을 고정한다.

## 1. 톤 방향 (증권사 기준 · 차분 / 저채도)

- **베이스:** 밝은 뉴트럴 그레이 배경 + 흰색 카드, 넉넉한 여백. 색이 거의 없는 차분한 바탕.
- **텍스트:** 순흑 대신 부드러운 차콜로 본문, 보조는 그레이. 고대비 유지(가독성·고령층 배려).
- **액센트:** 신뢰감 있는 딥블루 하나만 포인트(버튼·강조)로 사용. 원색·네온 배제.
- **상태색:** 성공/주의/오류 모두 채도를 낮춰 차분하게.
- **등락 색 (한국 증권 관례):** 상승 = 빨강, 하락 = 파랑, 보합 = 회색. 저채도 톤에 맞춰 너무 쨍하지 않게 적용. (※ 한국 증권 앱 표준이며 글로벌 관례인 "상승 = 초록"과 반대이므로, 상태색의 성공-초록과 의미가 겹치지 않게 구분)
- **형태:** 라운드 모서리(과하지 않게), 카드 기반, 그림자 최소 → 플랫하고 깔끔.
- **AI 파동(오브):** 전체가 차분하므로 오브가 유일한 색 포인트가 됨. 절제된 글로우로 두되 살짝 톤다운하여, 대비로 와우 포인트를 부각.

## 3. WaveIndicator (AI 파동) 시각 가이드

봇의 대기/듣는 중/말하는 중 상태를 시각적으로 표현하는 턴테이킹 신호. 데모에서 아래 두 형태를 모두 구현해 실제로 넣어보고 결정한다.

- **① 글로우 오브 (우선순위):** 분홍~파랑 글로우 구체. 직접 SVG+CSS 구현(라이선스 무관, 색·상태 완전 제어) 또는 Lottie 애셋 사용(특정 작가 애셋은 무료/상업권/출처 라이선스 확인 필수). 대기 = 은은한 호흡, 말하는 중 = 맥동.
- **② 시리식 파형:** SiriWave 라이브러리(kopiro/siriwave). 진폭을 상태에 따라 조절해 말하는 중/대기 표현.
- **공통:** 미관뿐 아니라 상태 구분(턴테이킹) 기능을 반드시 유지.
- 전체 톤이 차분하므로 오브가 유일한 색 포인트. 절제된 글로우로 두되 살짝 톤다운하여 와우 포인트를 부각(1장 톤 방향과 연계).

## 4. 화면 디자인 원칙

- **한 화면 한 과업:** 메인 콘텐츠 영역(zone 3)은 한 번에 하나의 과업만 노출.
- **접근성 우선:** 한 손 조작·주의 분산·고령층 사용 가능성을 전제로 **큰 터치 타깃·큰 글씨·고대비·적은 정보량**.
- **음성-화면 동조:** 봇이 말하는 항목을 화면에서 하이라이트하는 음성-화면 동조가 완성도의 핵심.

## 5. 반응형 전략 (모바일폰 + 태블릿 / PC 비대상)

- 모바일폰 **세로 화면**을 기준으로 설계.
- 태블릿은 콘텐츠 최대 폭을 제한해 가운데 정렬 + 여백을 넓히는 방식으로 대응(레이아웃 재설계 없이 한 디자인을 확장).
- PC는 대상에서 제외.

## 6. 디자인 주의사항 (금지 / 필수)

- **누가 봐도 AI가 만든 듯한 디자인 지양:** 천편일률적 템플릿 느낌을 피한다. (예: 과한 그라데이션·보라/청록 남발, 모든 요소 가운데 정렬, 똑같은 라운드 카드의 단조로운 나열, 의미 없는 아이콘 장식, 글래스모피즘 남용) → 증권사 실서비스에 어울리는 의도적이고 절제된 디자인을 추구.
